不容易被人看出规律（隐藏用户数量）, 散列好,做缓存字段更好处理, 不用自增降低数据库消耗，也更适合做分布式系统主键, 长度比 int 好, 字符串做主键也不会出现 long 型导致前端 js 麻烦问题

缺点也就性能差点

就算用自增主键，也可以另外生成一个带索引的 token 字段，隐藏用户 ID 和主键用什么，没有半毛钱关系
